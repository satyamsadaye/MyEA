Minimal Trade Copier Settings Guide

This document explains every setting in the copier EA, with practical examples for live use.

The copier reads open trades from the sender through the MetaTrader common files folder. Each sender position has a source ticket. The copier uses that source ticket in the copied trade comment, for example TC#53677970, so each copied trade can be managed separately.

Important Live Notes

  • Attach this EA on the receiving terminal only.
  • One attached copier can manage multiple mapped symbols when SymbolMode = SYMBOL_MULTI_PAIR.
  • The EA manages only trades with its own MagicNumber and comments starting with TC#.
  • If you change the source code, compile it again in MetaEditor and reload the EA on the chart.
  • If MT5 still shows old inputs like TakeProfitAdjustMode, or logs old text like risk money target, the terminal is still running an old compiled EA.
  • For live trading, test first on demo with the same broker symbols, account type, and lot step.

Signal Settings

CopierKey

Default: client1

This key chooses which sender file the copier reads.

The file name is:

minimal_trade_copier_<CopierKey>.csv

Example:

CopierKey=satyam

The copier reads:

minimal_trade_copier_satyam.csv

Use the same key on the sender and copier. If the keys do not match, the copier will not see the sender trades.

Allowed characters are letters, numbers, _, and -. Spaces and special characters are removed internally.

CopyMode

Values:

0 = COPY_EXACT_SAME
1 = COPY_EXACT_OPPOSITE

COPY_EXACT_SAME copies the same direction.

Example:

Sender BUY XAUUSD
Copier BUY XAUUSDm

COPY_EXACT_OPPOSITE flips the direction.

Example:

Sender BUY XAUUSD
Copier SELL XAUUSDm

With opposite copy, SL and TP are also reversed:

Sender BUY entry 4532.00
Sender SL 4529.00
Sender TP 4538.00

Copied SELL entry around market price
Copied SL = sender TP = 4538.00
Copied TP = sender SL = 4529.00

SymbolMode

Values:

0 = SYMBOL_SINGLE_PAIR
1 = SYMBOL_MULTI_PAIR

Use SYMBOL_SINGLE_PAIR when all sender signals should copy to one symbol or to the same symbol name.

Use SYMBOL_MULTI_PAIR when you want different sender symbols mapped to different copier symbols, for example XAUUSD -> XAUUSDm and EURUSD -> EURUSDm.

CustomSymbol

Used only when:

SymbolMode=0

If CustomSymbol is filled, every sender trade copies to that one symbol.

Example:

SymbolMode=0
CustomSymbol=XAUUSDm

Result:

Sender XAUUSD -> Copier XAUUSDm
Sender GOLD -> Copier XAUUSDm
Sender EURUSD -> Copier XAUUSDm

If CustomSymbol is empty, the copier uses the sender symbol name directly.

Example:

SymbolMode=0
CustomSymbol=

Result:

Sender EURUSD -> Copier EURUSD
Sender XAUUSD -> Copier XAUUSD

When SymbolMode=1, CustomSymbol is ignored. For clarity, leave it empty in multi-pair mode.

SenderSymbol1, CopierSymbol1

First symbol mapping pair. Used only when SymbolMode=1.

Example:

SenderSymbol1=XAUUSD
CopierSymbol1=XAUUSDm

Result:

Sender XAUUSD trades are copied to XAUUSDm.

SenderSymbol2, CopierSymbol2

Second symbol mapping pair.

Example:

SenderSymbol2=EURUSD
CopierSymbol2=EURUSDm

Result:

Sender EURUSD trades are copied to EURUSDm.

SenderSymbol3, CopierSymbol3

Third symbol mapping pair.

Leave both empty if you do not need a third pair.

Example:

SenderSymbol3=
CopierSymbol3=

If the sender sends a symbol that is not configured in multi-pair mode, the copier skips it and shows a message like:

Skipping source #123456 because sender symbol GBPUSD is not configured

Entry Settings

EntryMode

Values:

0 = COPIER_ENTRY_MARKET_ORDER
1 = COPIER_ENTRY_PENDING_ORDER

COPIER_ENTRY_MARKET_ORDER opens immediately at current copier market price.

Example:

Sender BUY opens at 4532.00
Copier sees signal at current price 4532.48
Copier opens immediately at 4532.48

COPIER_ENTRY_PENDING_ORDER places a pending order at the sender entry price.

Example:

Sender entry = 4532.00
Copier current price = 4535.00
Copier places pending order around 4532.00 if broker rules allow it.

For live copying, market mode is simpler. Pending mode can be useful when you want the copier to enter closer to the sender price, but the order may not be valid if the price is too close to current market price or on the wrong side for the pending type.

CopyStopLossTakeProfit

Values:

false = do not copy SL/TP
true = copy SL/TP

When true, the copier sends SL and TP with the copied trade.

In same copy:

Copied SL = sender SL
Copied TP = sender TP

In opposite copy:

Copied SL = sender TP
Copied TP = sender SL

For fixed-risk lot sizing, this should normally stay true, because the EA needs an SL distance to calculate lot size.

AdjustTakeProfitToCopiedRR

Values:

false = keep copied TP as directly mapped
true = adjust TP after fill to preserve copied RR

When true, the copier recalculates TP after the actual copied entry price is known.

This matters because market entry may not match the sender entry exactly. If the copied trade enters at a slightly different price, the original copied TP may no longer represent the intended risk/reward.

Current behavior:

The EA preserves copied RR.
It does not force 1:1.
It does not target fixed money profit.

Example, opposite copy:

Sender BUY
Entry 4532.72
SL 4529.41
TP 4539.23

Sender risk:

4532.72 - 4529.41 = 3.31

Sender reward:

4539.23 - 4532.72 = 6.51

Sender RR is about:

6.51 / 3.31 = 1.97R

With opposite copy:

Copied SELL
Copied SL = 4539.23
Copied TP = 4529.41

Copied risk distance:

4539.23 - 4532.72 = 6.51

Copied reward distance:

4532.72 - 4529.41 = 3.31

Copied RR is:

3.31 / 6.51 = 0.51R

So if fixed risk is about $100, the opposite copied target is about:

$100 * 0.51 = $51

That is correct for exact opposite copied RR. It is not a bug. If sender has 2R win and 1R loss, the opposite trade naturally has about 0.5R reward against 1R risk.

Sender Exit Settings

SenderExitAction

Values:

0 = SENDER_EXIT_CLOSE_ALWAYS
1 = SENDER_EXIT_PROTECT_ALWAYS
2 = SENDER_EXIT_SMART_PROTECT_PROFIT_CLOSE_LOSS

The sender file contains currently open sender trades. When a sender trade disappears from the file, the copier treats that as sender exit.

SENDER_EXIT_CLOSE_ALWAYS

Always closes the copied trade when the sender trade closes.

Example:

Sender trade closes
Copied trade closes immediately

Use this if you want the copied trade to never continue after sender exit.

SENDER_EXIT_PROTECT_ALWAYS

When sender exits, the copier tries to move SL to breakeven or profit lock.

If protection is not possible, it closes the copied trade.

Example:

Copied SELL entry 4532.00
Current price 4528.00
Sender exits
Copier moves SL to breakeven or profit lock if broker allows it

SENDER_EXIT_SMART_PROTECT_PROFIT_CLOSE_LOSS

This is your current smart mode.

Behavior:

If copied trade is profitable when sender exits: protect it.
If copied trade is not profitable when sender exits: close it.

This is useful for opposite copying.

Example with opposite copy:

Sender BUY hits SL.
Copied SELL is likely profitable.
Copier protects the profitable SELL.

Example:

Sender BUY hits TP.
Copied SELL is likely losing.
Copier closes the losing SELL.

Important limitation:

The copier does not know the exact reason the sender closed unless the sender publishes closed-history information. It infers from the copied trade's current profit/loss.

SenderExitProfitLockPercent

Range used by the EA:

0 to 100

0.0 means move SL to breakeven.

Example:

SenderExitProfitLockPercent=0.0

If copied SELL entry is 4532.00 and current price is 4528.00, the EA tries to move SL near 4532.00.

50.0 means lock 50 percent of open profit.

Example:

Copied SELL entry 4532.00
Current price 4528.00
Open profit distance = 4.00
50 percent lock = 2.00
New SL = 4530.00

Broker stop limits may prevent the EA from moving SL exactly where requested.

Lot Sizing

LotMultiplier

Used only when:

LotMode=0

Formula:

Copied lot = sender lot * LotMultiplier

Example:

LotMode=0
LotMultiplier=2.0

If sender opens 0.10, copier opens:

0.10 * 2.0 = 0.20

LotMode

Values:

0 = COPIER_LOT_EXACT
1 = COPIER_LOT_FIXED_STARTING_BALANCE_RISK

COPIER_LOT_EXACT copies sender volume using LotMultiplier.

COPIER_LOT_FIXED_STARTING_BALANCE_RISK calculates lot from a fixed account size and risk percent.

RiskStartingBalance

Used only when:

LotMode=1

This is the balance used for risk calculation. It does not automatically change with account equity.

Example:

RiskStartingBalance=5000.0

RiskPerTradePercent

Used only when:

LotMode=1

Formula:

Risk money = RiskStartingBalance * RiskPerTradePercent / 100

Example:

RiskStartingBalance=5000.0
RiskPerTradePercent=2.0

Risk money:

5000 * 2 / 100 = 100

The EA then calculates the lot size that should lose about $100 if the copied SL is hit.

Fixed Risk Example

Log example:

Risk Sizing: Entry=4532.63300 | SL=4539.23000
LossPer1Lot=659.70
RawLot=0.15158405
NormalizedLot=0.15
ExpectedRisk=98.95

Meaning:

Risk money target = $100
If 1.00 lot hits SL, loss is about $659.70
Raw lot = 100 / 659.70 = 0.1515
Broker lot step rounds it to 0.15
Expected risk = about $98.95

MaxFixedRiskLot

Used only when:

LotMode=1

This is a safety cap. If calculated fixed-risk lot is above this value, the EA skips the trade.

Example:

MaxFixedRiskLot=0.50

If fixed-risk calculation says 0.80, the EA skips the trade.

Why this matters:

If SL distance is very small, the calculated lot can become very large. This setting prevents oversized live trades.

LogRiskSizingDetails

Values:

false = do not print detailed risk sizing logs
true = print detailed risk sizing logs

Recommended for testing and early live use:

LogRiskSizingDetails=true

The logs help confirm entry price, SL, risk money, raw lot, normalized lot, and expected risk.

Trade Execution

MagicNumber

The EA manages only positions and pending orders with this magic number and TC# comments.

Example:

MagicNumber=991

Use a different magic number if you run multiple copier instances on the same terminal.

Do not use the same magic number for unrelated EAs.

SlippagePoints

Maximum allowed slippage/deviation in points for market trade actions.

Example:

SlippagePoints=30

Higher value means the broker has more room to fill during fast movement. Lower value means more rejected trades.

PollMilliseconds

How often the copier checks the sender file.

Example:

PollMilliseconds=500

This checks about twice per second.

Very low values can make the terminal busier. The EA enforces a minimum of 100.

TradeRetries

Number of retries for trade operations.

Example:

TradeRetries=5

If a close, open, modify, or delete fails temporarily, the EA retries.

RetryDelayMilliseconds

Delay between retry attempts.

Example:

RetryDelayMilliseconds=250

With TradeRetries=5, the EA may spend about one second retrying a failing operation.

Multi-Symbol Management

Yes, the copier can manage two symbols without conflict when configured correctly.

Example:

SymbolMode=1
SenderSymbol1=XAUUSD
CopierSymbol1=XAUUSDm
SenderSymbol2=EURUSD
CopierSymbol2=EURUSDm

If sender sends:

Ticket 1001 XAUUSD BUY
Ticket 1002 EURUSD SELL

The copier manages:

TC#1001 on XAUUSDm
TC#1002 on EURUSDm

It separates trades by:

source ticket
symbol
direction
magic number
comment

So the XAUUSDm trade will not be treated as the EURUSDm trade.

Duplicate Re-Entry Protection

The EA records when a source ticket has already started a copied lifecycle.

If the copied trade closes while the sender trade is still open, the EA will not reopen that same source ticket again.

Example:

Sender ticket #53677970 is still open.
Copier opens TC#53677970.
Copied trade hits SL.
Sender is still open.
Copier sees #53677970 again.
Copier does not re-enter.

This prevents repeated entries from the same sender trade.

The lifecycle marker is stored in MT5 terminal Global Variables using a name like:

MTC1_<MagicNumber>_<CopierKey>_<SenderLogin>_<SourceTicket>

For testing the exact same sender ticket again, you may need to delete that global variable manually in MetaTrader.

Frozen Trade Handling

Sometimes MT5 returns:

retcode 10029 frozen

This means the broker temporarily refuses to close or modify that position, often because price is too close to stop/freeze levels or the symbol is in a restricted state.

Current EA behavior:

If an old orphan copied trade cannot close/protect, the EA waits.
It does not open new sender signals until that old copied trade is handled.

This prevents stacked exposure when an old copied trade is stuck.

Before Going Live Checklist

  1. Confirm the sender and copier use the same CopierKey.
  2. Confirm exact broker symbol names, for example XAUUSDm, EURUSDm.
  3. Use SymbolMode=1 for multiple symbols.
  4. Leave CustomSymbol empty in multi-pair mode.
  5. Use a conservative MaxFixedRiskLot.
  6. Keep LogRiskSizingDetails=true during first live tests.
  7. Confirm chart dashboard shows correct key, mapping, and magic number.
  8. Confirm copied trade comments are TC#<sender ticket>.
  9. Confirm logs say for copied RR, not old risk money target.
  10. Test one small trade first before normal live size.
